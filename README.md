How do I approach designing software?
  -What new design skills has your work on the project helped you to craft?
  
  I have come to realize that my artistic vision still needs some work. Working with these 3d models gave me some freedom to be creative and desgin something I wanted to design. While this project did help improve this, I still need to work on how I can represent art with my code, or image manipulation. These will be skills that I will need to improve on in the field of Computer Science, whether for web-page design, game creation, or UI development.
  
  -What design process did you follow for your project work?

  For this project I took down all requirements and tackled one at a time. Implementing each part sequentially helps keep the program organized and reduces necessary testing later on. Having code that compiles for the entire project is achievable this way, and I should not have to tell you how sucessful that can make coding an entire project. Dealing with bugs as they appear, rather than postponing them minimizes the endles searching for fixes when they pile up.
  
  -How could tactics from your design approach be applied in future work?

  I hope to continue this approach to software design as it echoes parts of the agile software development lifecycle and helps me keep track of my progress during the project. Programming this was helps keep me focused during programming as I always have a next step to be working towards, making hours of coding seem like small steps towards a large end goal. Having a set of requirements separate from the project can help with nearly every project I will work on. Not only does it help ensure that all requirements are complete, but it also helps keep track of the entire project for when I need to pick it back up after a while off. It also helps show the entire scope of the project ensuring that every part is captured during development.
